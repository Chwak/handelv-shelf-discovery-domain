HANDLER: update-search-index-lambda.ts

PRIMARY LOGIC
Resolves traceparent from headers or generates one (currently unused beyond assignment). Validates shelfItemId or alias productId argument. action string default upsert lowercased — delete issues DeleteCommand by key shelfItemId; else merges args.document object into Item with shelfItemId, productId fallback, updatedAt ISO, Put to SEARCH_DOCUMENTS_TABLE. Returns true.

PROS
- Single-call upsert/delete for auxiliary search documents.

CONS
- **No authentication/authorization in code** — anyone who can invoke the resolver can mutate search documents unless AppSync/IAM strictly limits callers.
- Spreads arbitrary document fields from client without schema validation in handler.

SECURITY
HIGH RISK if exposed broadly: needs admin/maker checks matching product ownership.

PERFORMANCE
One Dynamo write/delete; cheap.

FURTHER IMPROVEMENTS
Enforce caller role; validate allowed attributes; strip unknown keys; use condition expressions for version control.

EXPOSES SENSITIVE INFORMATION
NO direct leak; malicious caller could inject misleading public search fields.

USER-FACING
YES if wired to GraphQL — should be privileged pipeline only.

EDGE CASES
PARTIAL — missing table throws; empty document upserts mostly-empty item with keys.
